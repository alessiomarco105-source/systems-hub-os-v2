# Learning Capture

Agents should propose capability updates when a task produces reusable knowledge.

## When To Capture

- a validation failure teaches a better output contract;
- a repeated manual workflow should become a script or checklist;
- a domain pattern improves future decisions;
- an agent discovers a better source or method;
- a bug or operational failure reveals a prevention rule.

## Where To Capture

Use `capabilities/skill-update-template.md`.

Place proposed updates inside the relevant capability folder, for example:

- `capabilities/design/`
- `capabilities/finance/`
- `capabilities/security/`
- `capabilities/marketing/`

## Promotion

A proposed skill is not active authority until reviewed.

Promotion requires:

- source named;
- owner named;
- reviewer named;
- risks stated;
- validation method stated;
- no conflict with governance, security, model access, or Marco preferences.
