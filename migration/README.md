# Migration

The legacy workspace remains operational while v2 is built in parallel.

## Classifications

| Action | Meaning |
|---|---|
| migrate | Move reviewed canonical content into the target |
| consolidate | Merge overlapping sources into one new canonical artifact |
| adapter | Rebuild from canonical source as platform-specific config |
| private | Keep outside normal model-readable repositories |
| generated | Exclude from canonical knowledge; regenerate when needed |
| archive | Preserve as historical evidence, excluded from routine context |
| delete-review | Likely disposable, but requires review before deletion |
| keep-legacy | Keep operational until v2 replacement passes |

`inventory.csv` and `inventory-summary.md` are generated by:

```bash
node systems-hub-os-v2/runtime/scripts/generate-migration-inventory.mjs
```

Generation reads paths and metadata only. It does not inspect or print secret values.
